# Core Loop

This document was created with the assistance of an AI.

## The day/night alternation
- **Day:** Champion Phase
  - direct control, exploration, building, combat, leadership decisions
- **Night:** Deity Phase
  - miracle selection, pantheon diplomacy/conflict, influence allocation

### Design goal
Each phase must:
- create problems the other phase can address,
- and create consequences the other phase must live with.

## Loop cadence (tunable)
- Default: 1 day = 12–20 minutes real-time (tunable).
- Nights are shorter but denser (5–10 minutes), focused on decisions and consequences.
- Long-term arc: “seasons” or “eras” that shift threats and pantheon politics.

## Inputs/outputs between phases
Champion → Deity:
- witnessed events (what happened today)
- settlement conditions (scarcity, fear, morale)
- relationships/trust shifts
- chaos level around Gates
Deity → Champion:
- miracles enacted (boons/banes)
- pantheon state changes (alliances, interdictions)
- world-rule mutations (temporary or permanent)
- new omens/quests/constraints

## Failure loops (intended)
- Overusing miracles increases Chaos attention.
- Ignoring settlement social dynamics makes miracles less believable.
- Chaos intrusion increases need for miracles → risks runaway escalation.

## What “winning” means (initial)
Not a single win condition—multiple victory narratives:
- stabilize the Gates for an era
- ascend within pantheon rules
- create a resilient society that survives Chaos cycles
- complete mythic arcs (Prometheus/Pandora variations)
