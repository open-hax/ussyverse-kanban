# Frontend SPA Issue Plan

## Context

- Goal: Identify initial GitHub issues to build a web SPA for knowledge graph visualization and editing.
- Backend library exists; no web UI in repo.

## Proposed Issue Themes

1. Graph API layer for SPA (REST/GraphQL) to query nodes/edges and mutations.
2. Web SPA shell with graph visualization (Cytoscape/Mermaid), search, filtering, and detail panels; visibly distinguish embedding/semantic edges with a dedicated color.
3. Node/edge management UI (create/edit/delete) wired to API, with validation and optimistic updates.
4. Build/deploy tooling for SPA (bundler, lint/test/CI, hosting target).

## Definition of Done

- Issues created in GitHub with clear scope, acceptance criteria, and links to relevant code (if any).
